# v0.1 Product Acceptance Pack

Active acceptance pack.

This is the explicit gate for the current v0.1 product slice.
It defines what the project must be able to demonstrate right now, using real
artifacts and one canonical focused verification command.

## Product Slice

Current v0.1 product slice:

- `PDF -> accessible PDF`
- `PDF -> accessible HTML`
- honest `needs_review` stop when the workflow cannot safely finalize
- operator loop for review decisions and OCR recovery on complex PDFs

## Acceptance Checklist

The product slice is real only if all of the following are true.

### 1. Simple Born-Digital PDF Finalizes End To End

Required outcome:

- `pdf_accessibility.cli run` finishes with:
  - `document_status: finalized`
  - `finalization_state: finalized`
- run directory contains:
  - `tagged_draft.pdf`
  - `accessible_output.pdf`
  - `accessible_output.html`
  - `operator_guide.json`
- CLI output ends with:
  - `primary_next_step: none`

Reference artifact:

- [abm-simple-ready_20260426T012657Z](/private/tmp/abm-simple-ready-runs/abm-simple-ready_20260426T012657Z:1)

### 2. Simple List HTML Output Is Credible

Required outcome:

- finalized simple-list run emits:
  - heading as `<h1>`
  - flat list items as `<ul>/<li>`
- this proves the supported simple structure slice is credible in HTML, not
  only in tagged PDF writeback

Required outcome:

- untreated `finreport25` run finishes with:

  - `finalization_state: needs_review`
- run directory contains:
  - `review_decisions_template.json`
  - `ocr_recovery_template.json`
  - `operator_guide.json`
- operator guide points at the next action without reading code

- reduces blockers materially without pretending to finalize

### 5. The CLI Entry Point Surfaces The Next Action

Required outcome:

- `pdf_accessibility.cli run` prints:
  - `run_dir`

  - `finalization_state`
  - `blocker_count`
  - `review_decisions_template`
  - `ocr_recovery_template`
  - `primary_next_step`

## Current Scope Boundaries

The acceptance pack does not require:

- complex table finalization
- full figure completion on `finreport25`
- link/annotation writeback
- arbitrary complex PDF finalization

Those remain outside the current v0.1 gate.

## Pass/Fail Rule

Pass means:

- the simple finalized path works
- the blocked complex path works
- the operator loop materially improves a real complex document
- the terminal and run artifacts tell a human what to do next

Fail means any of these regress:

- finalized simple runs stop finalizing
- blocked runs lose their honest next-step guidance
- OCR/review reruns stop reducing real blockers
- HTML output regresses below the supported simple structure slice
