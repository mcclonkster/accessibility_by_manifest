# Shared project files

Use this file set to keep planning, tracking, research, and handoff separate.

## The file set

- `INTAKE.md`
- `PLAN.md`
- `STATUS.md`
- `TASKS.md`
- `DECISIONS.md`
- `RESEARCH-PLAN.md`
- `FINDINGS.md`
- `HANDOFF.md`

## What each file is for

### `INTAKE.md`
Use for new incoming work before it becomes part of the active project.

Good for:
- new requests
- bugs
- feature ideas
- cleanup ideas
- decisions needed
- research requests

Do not use it for:
- detailed project planning
- live task tracking
- research notes
- handoff summaries

### `PLAN.md`
Use for the actual project plan.

Keep:
- purpose
- scope
- constraints
- assumptions
- milestones
- acceptance criteria
- risks
- dependencies
- next checkpoint

Do not turn it into:
- a task log
- a research notebook
- a general scratchpad

### `STATUS.md`
Use for the current high-level state of the project.

Keep:
- current phase
- current health
- what changed recently
- blockers
- open questions
- next checkpoint

Do not use it for:
- full planning detail
- full task detail
- raw research output

### `TASKS.md`
Use for live execution tracking.

Keep:
- active
- blocked
- waiting
- next
- done
- resume notes

Do not use it for:
- milestone planning
- scope changes
- deep reasoning about project direction

### `DECISIONS.md`
Use for decisions that materially affect the project.

Keep:
- decision
- rationale
- consequence
- decisions still needed

Do not use it for:
- temporary task status
- raw research notes
- implementation details that do not change project direction

### `RESEARCH-PLAN.md`
Use when the project needs explicit research before a decision or planning update.

Keep:
- research question
- why it matters
- scope
- exclusions
- source types
- evidence threshold
- open uncertainties

Do not use it for:
- final research findings
- project planning
- live task tracking

### `FINDINGS.md`
Use for research output.

Keep:
- source-grounded findings
- synthesis
- conflicts
- unknowns
- what planning should know next

Do not use it for:
- raw intake
- project planning
- live task lists

### `HANDOFF.md`
Use for resuming the project later.

Keep:
- what the project is
- what is current
- what changed in the last pass
- what still needs resolution
- what to do first next time
- traps or caveats

Do not use it for:
- full project duplication
- exhaustive logs
- raw research dump

## The usual flow

1. Capture new work in `INTAKE.md`.
2. Move accepted work into `PLAN.md`.
3. Reflect current project state in `STATUS.md`.
4. Track live execution in `TASKS.md`.
5. Record material direction changes in `DECISIONS.md`.
6. If research is needed, define it in `RESEARCH-PLAN.md`.
7. Record research output in `FINDINGS.md`.
8. Leave the next session a clean restart point in `HANDOFF.md`.

## Boundary rules

- `INTAKE.md` is for incoming work, not committed project structure.
- `PLAN.md` defines the work.
- `STATUS.md` says where things stand.
- `TASKS.md` tracks what is happening right now.
- `DECISIONS.md` records what has been decided and why.
- `RESEARCH-PLAN.md` defines what needs to be investigated.
- `FINDINGS.md` records what the investigation found.
- `HANDOFF.md` makes resumption easy.

## The main goal

Someone should be able to open these files and answer:

1. What is this project?
2. What are we doing?
3. What is happening right now?
4. What still needs research?
5. What was learned?
6. Where do we pick up next?
