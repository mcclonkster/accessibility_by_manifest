# Project Control Files

This directory is the current canonical planning and control surface for the repository.

Active minimum set:

1. [system reference](../docs/design/system_reference.md)
2. [workflow rules](../AGENTS.md)
3. [repo guide](../README.md)
4. [product plan](../plans/start-to-finish-product-plan.md)
5. `STATUS.md`
6. `TASKS.md`
7. `DECISIONS.md`

Use it for:

- status
- task tracking
- decisions
- cleanup review files

Start here in this order:

1. `STATUS.md`
2. `TASKS.md`
3. `DECISIONS.md`

Additional files:

- `QUARANTINE/` - manual cleanup decision queue
- `CLEANUP-REPORT.md` - current cleanup assessment

Frozen or quarantined guidance lives under `QUARANTINE/guidance/`.

Notes:

- `AGENTS.md` remains the governing source for current v0.1.0 PDF workflow constraints.
- `project-file-structure-md/` is canonical until a later explicit move to the repo root is decided and performed deliberately.
