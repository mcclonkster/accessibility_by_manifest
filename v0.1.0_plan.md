Below is the practical **v0.1.0 plan**.

**Goal**

v0.1.0 should take a born-digital PDF and produce an honest accessibility workflow:

```text
input PDF
-> evidence
-> normalized structure / IR
-> review tasks
-> review decisions
-> structure mapping
-> tagged draft PDF
-> validation
-> finalization status
```

For real reports, `needs_review` is acceptable. v0.1.0 should not pretend the output is final unless it is actually legal to finalize.

**Phase 1: Workflow Spine**

Status: mostly done.

1. Create run folders.
2. Copy source PDF as `input.pdf`.
3. Write required artifacts:
   - `document_state.json`
   - `findings.jsonl`
   - `review_tasks.json`
   - `review_decisions.json`
   - `normalized_structure.json`
   - `structure_mapping_plan.json`
   - `tagging_plan.json`
   - `validator_findings.json`
   - `finalization_status.json`
   - `artifact_manifest.json`
   - `run_log.json`
4. Route nodes through shared state.
5. Record which nodes ran or were skipped.

**Phase 2: Deterministic Evidence**

Status: partially done.

1. Extract document metadata:
   - title
   - language
   - page count
   - PDF version
   - encryption state
2. Extract page geometry.
3. Extract text blocks, spans, fonts, font sizes, bounding boxes.
4. Extract images.
5. Extract links and annotations.
6. Add stronger pikepdf/pypdf/pdfminer evidence later where needed:
   - MarkInfo
   - StructTreeRoot
   - MCIDs
   - outlines
   - resources
   - low-level object refs

**Phase 3: Normalization / IR**

Status: biggest missing layer.

1. Create a stable intermediate representation.
2. Classify page types:
   - cover
   - narrative
   - table-heavy
   - appendix
   - mixed
3. Detect page regions:
   - header
   - footer
   - page number
   - body
   - table
   - figure
   - caption
4. Normalize text blocks into:
   - headings
   - paragraphs
   - lists
   - list items
   - tables
   - figures
   - artifacts
   - unknowns
5. Assign reading order.
6. Preserve provenance back to source evidence.
7. Add confidence levels.

**Phase 4: Review Tasks**

Status: started, needs refinement.

1. Generate review tasks only for meaningful uncertainty.
2. Required task types:
   - missing title
   - missing language
   - uncertain heading
   - uncertain table headers
   - uncertain table structure
   - figure alt text required
   - image-only page review
   - reading order review
3. Separate blocking from non-blocking tasks.
4. Avoid dumping everything into review.

**Phase 5: Review Decision Loop**

Status: next major step.

1. Write a usable `review_decisions.json` template.
2. Let reviewer decisions resolve simple blockers:
   - set document title
   - set primary language
   - mark figure decorative
   - provide alt text
   - resolve safe review task
3. Re-apply decisions.
4. Rerun structure mapping when decisions change metadata or structure readiness.
5. Keep non-resolvable blockers blocked:
   - draft not final
   - unsupported writeback
   - complex table unresolved

**Phase 6: Structure Mapping**

Status: early.

1. Map normalized units to PDF structure roles:
   - heading -> `H`
   - paragraph -> `P`
   - list -> `L`
   - list item -> `LI`
   - table -> `Table`
   - figure -> `Figure`
   - artifact -> excluded/artifact
2. Plan MCIDs.
3. Plan ParentTree entries.
4. Mark what can be written safely.
5. Mark what is unresolved.

**Phase 7: Tagged Draft PDF**

Status: tiny/simple cases only.

1. Write document title.
2. Write primary language.
3. Add MarkInfo.
4. Add StructTreeRoot.
5. Add marked content for simple H/P cases.
6. Add ParentTree when MCIDs are written.
7. Produce `tagged_draft.pdf`.
8. Clearly mark it as draft, not final.

**Phase 8: Validation**

Status: internal validator started; veraPDF later.

1. Run internal checks:
   - PDF opens
   - has title
   - has language
   - has MarkInfo
   - has StructTreeRoot
   - marked content matches writeback report
   - ParentTree exists when MCIDs exist
2. Write `validator_findings.json`.
3. Later add optional veraPDF.

**Phase 9: Finalization Gate**

Status: started.

1. Finalize only if:
   - tagged draft exists
   - no unresolved blockers
   - validator passes
   - review decisions are applied
2. If legal, write `accessible_output.pdf`.
3. If not legal, stop at:
   - `needs_review`
   - or `write_blocked`
4. Never silently call a draft final.

**Phase 10: Optional AI Sidecars**

Status: later, not core yet.

Add after the deterministic evidence/IR/review loop is stable.

AI should help with:

- alt-text drafts
- table boundary hints
- chart summaries
- reading-order comparison
- scanned/image-heavy fallback
- artifact detection hints

AI should not replace the deterministic manifest, normalizer, or review gate.

**Recommended Build Order From Here**

1. Review decision loop.
2. Better normalization/IR.
3. Better artifact/header/footer detection.
4. Simple table detection.
5. Figure/caption/alt-text planning.
6. Stronger structure mapping.
7. Better tagged draft writer.
8. Internal validator improvements.
9. veraPDF integration.
10. Optional AI sidecars.

**TL;DR**

v0.1.0 is not “perfect accessible PDF generation.” It is a reproducible local workflow that can extract evidence, normalize structure, create review tasks, accept review decisions, write a limited tagged draft, validate it, and honestly stop unless finalization is justified.
